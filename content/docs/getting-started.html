Sign up for an AWS account

There are no upfront charges or any term commitments to create an AWS account.

Provision Cluster

AWS Controller can run on any cluster whether it's eksctl, Amazon EKS, Kind, Minikube or any other.

  • • You need is an IAM Role with AdministratorAccess
					
export NAME=awsctrl
eksctl create cluster --name=${NAME}
eksctl utils associate-iam-oidc-provider --cluster=${NAME} --approve
eksctl create iamserviceaccount --cluster=${NAME} --name=default --namespace=awsctrl-system --attach-policy-arn=arn:aws:iam::aws:policy/AdministratorAccess  --approve

Install awsctrl CLI

The awsctrl CLI allows you to install and manage the AWS Controller installed in your clusters.

  • • Install Golang® if not already on your machine.
					
go install go.awsctrl.io/manager/cmd/awsctrl

Install awsctrl

Once you have a cluster you can install it into your cluster using the awsctrl CLI.

					
awsctrl install manager | kubectl apply -f -
awsctrl install config | kubectl apply -f -

Create AWS Resources

After AWS Controller is running you can deploy AWS Resources like this example ECR Repository.

					
cat >ecr.yaml <<EOF
apiVersion: ecr.awsctrl.io/v1alpha1
kind: Repository
metadata:
  name: repository-sample
spec: 
  repositoryName: awsctrl-test
EOF
kubectl apply -f ecr.yaml
